Raspberry Pi Setup

July 02, 2020
Raspberry Pi

My Raspberry Pi setup is constantly changing. They are my mini homelab. Used when I want containers to run on a more permanant basis. All average at about 3W in power, making them very efficient to their tasks. Most have Docker installed and are blocked from access to the internet if hosting private information.

Tech setup

  • Original Pi B (Kept for mainly sentimental)
  • Pi 3 - Potential Home Assistant server
  • Pi 3 B+ - This website
  • Pi 4 (4GB) - PiVPN and Nextcloud

Website

This website is hosted using Nginx and a reverse proxy. Built using Hugo using markup language.

Docker

Docker is a container system that I understand with great detail. There are a few aspects that I could learn but I haven’t had the need to learn them. I understand the virtualised networks and how to expose them as well.

Currently Docker is used to host Gitlab and Nextcloud. In the future this website will be in a Docker container and Home Assistant will be placed on a different Pi.

Nextcloud

In oder to preserve security Nextcloud is designed to be accessed through a reverse proxy to reduce security risks it hosts family documents and is designed to be as secure as possible. Thus hosting on an entirely seperate Pi. Also having a dedicated Pi for processing files helps with bottlenecks.
